# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Overview

This is **mathlib4**, the community-maintained mathematics library for the Lean 4 theorem prover. It contains formalized mathematics, programming infrastructure, and tactics. The toolchain is specified in `lean-toolchain` (currently Lean 4.29.0-rc1).

## Essential Commands

```bash
# Download precompiled .olean files (ALWAYS run first, otherwise builds take hours)
lake exe cache get

# Build entire library
lake build

# Build a specific file (use module path, not file path)
lake build Mathlib.Algebra.Group.Defs

# Run all tests
lake test

# Text-based style linting
lake exe lint-style

# Regenerate import files (Mathlib.lean, etc.) after adding new .lean files
lake exe mk_all
```

If builds behave strangely, try `lake clean` or `rm -rf .lake` then `lake exe cache get` again.

## Project Structure

- **Mathlib/** — Main library organized by mathematical area (Algebra, Analysis, CategoryTheory, Topology, MeasureTheory, etc.). `Mathlib/Tactic/` contains custom tactics and linters.
- **Mathlib.lean** — Auto-generated root import file (regenerated by `lake exe mk_all`)
- **Archive/** — Standalone formalizations of specific results (e.g., olympiad problems)
- **Counterexamples/** — Formalized counterexamples
- **MathlibTest/** — Test suite for tactics and linters
- **Cache/** — Implementation of the `lake exe cache` tool
- **scripts/** — Build, linting, and CI scripts (Lean and Python)
- **docs/** — Documentation-only modules

## Key Conventions

- **`autoImplicit` is `false`** — All variables must be explicitly declared.
- **`pp.unicode.fun` is `true`** — Code uses `fun a ↦ b` notation (not `fun a => b`).
- **`Mathlib.Init` must be imported** — Enforced by linter; all Mathlib files transitively import it.
- **File length limit: 1500 lines** — Enforced by `linter.style.longFile`. Split files if they grow too large.
- **Line width: 100 columns** — Per style guide.
- **2-space indentation**, UTF-8 encoding, LF line endings, trailing whitespace trimmed.

## Style and Naming

Follow these guides (enforced by CI):
- [Style guide](https://leanprover-community.github.io/contribute/style.html)
- [Naming conventions](https://leanprover-community.github.io/contribute/naming.html) — Theorem names mirror their statements (e.g., `List.map_cons`, `Nat.add_comm`).
- [Documentation style](https://leanprover-community.github.io/contribute/doc.html)

## Linting

Mathlib has extensive automated linting. The `mathlibOnlyLinters` in `lakefile.lean` define the full set. Key linters:
- `linter.mathlibStandardSet` — Standard Mathlib linter suite (runs during `lake build`)
- `linter.style.header` — Module docstring and copyright header requirements
- `lake exe lint-style` — Text-based style checks (run separately)

## Architecture Notes

- **lakefile.lean** defines the package configuration, dependencies, lean options, and all executables.
- **Dependencies**: batteries, Qq, aesop, ProofWidgets (pinned to specific version), importGraph, LeanSearchClient, plausible.
- **`lake exe cache`** is critical infrastructure — it manages downloading/uploading precompiled .olean files. Without it, building from scratch takes many hours.
- **ProofWidgets is pinned** to a specific version tag. If you see errors about ProofWidgets, run `lake exe cache get`.
- **`lake exe mk_all`** scans directories and regenerates the import-all files (`Mathlib.lean`, `Archive.lean`, `Counterexamples.lean`). Must be run after adding or removing `.lean` files.
